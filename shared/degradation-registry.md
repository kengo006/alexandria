# Degradation registry (template)

A machine-readable, honest list of text-layer files that have been **repaired to their ceiling and are still not grep-reliable**. The registry exists because of one principle:

> **A file you could not fully fix is registered, never hidden.** Hiding it turns every future grep miss on that file into a confident, wrong "it's not in the vault."

Some sources never yield a clean text layer — a hostile scan, a font with a private encoding, a language your OCR mishandles. Repair what can be repaired (see `roles/librarian.md` §8); what remains gets a registry entry stating exactly **which access paths still work**. The searching roles consult the registry before shipping any negative conclusion (`roles/searcher.md` §1).

## Schema

One JSON file, regenerated by the Librarian whenever a repair batch lands — a **living file**: consumers read it fresh, never cache it, never copy its numbers elsewhere.

```json
{
  "version": 2,
  "generated": "2026-07-20T12:00:00",
  "generator": "corpus QA pipeline (see roles/librarian.md §8)",
  "count": 2,
  "files": [
    {
      "rel": "memory-studies/Doe_2011_Archives of Memory.txt",
      "why": "ceiling — hostile scan; diacritics collapse under OCR",
      "usable": { "semantic": true, "grep": false },
      "note": "grep is unreliable on this file (0 hits != absent); semantic recall and page rendering still work"
    },
    {
      "rel": "memory-studies/Roe_1998_Commemoration and Its Discontents.txt",
      "why": "fixing — residual word-gluing, re-OCR queued",
      "usable": { "semantic": true, "grep": false },
      "note": "temporary entry; drops off the registry when the repair batch lands"
    }
  ]
}
```

*(Both rows are illustrative placeholders — replace with your own.)*

## Iron rules

- **Every entry carries its usable paths.** "Degraded" never means "unusable" — it means *which* access path lies. A file whose grep lies but whose semantic recall works is still a working source; the registry is what keeps it one.
- **`count` must equal the actual row count.** A generator that writes a count it didn't measure is manufacturing a credential; consumers that detect a mismatch report the measured value and flag the discrepancy — they do not silently pick one.
- **Entries are states, not verdicts.** `fixing` entries drop off when their repair lands; `ceiling` entries stay until the source itself is replaced. Regenerate, don't hand-edit.
- **The searching roles' obligation is mechanical**: before any "not in the corpus / nowhere in this work" conclusion, check the file against the registry. Listed = grep silence is not evidence; use the paths marked usable.

## The index self-report (closing the loop)

A registry only the humans read will be skipped. Wire it into the search index itself: the index's info endpoint reports the degraded count, so a searching agent learns *from the tool it is already holding* that some files' grep silence means nothing. The contract lives in `optional-integrations.md` §2; its one non-negotiable:

> **Unreadable registry → `status: "unknown"`, never `0`.** "Zero degraded files" asserts that negative conclusions are safe everywhere; "I could not read the registry" asserts nothing. Collapsing the second into the first makes the agent boldest exactly when it should be most careful.
